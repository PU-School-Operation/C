# C語言圖書管理系統 (Library Management System) - 開發規格與需求書

## 1. 系統概述

本專案為一個基於 C 語言的命令列介面（CLI）圖書管理系統。系統需具備帳號登入驗證、圖書增刪查改、借還書管理等功能。系統啟動時需從預設的純文字檔載入資料，並在執行期間維護資料的狀態 。

## 2. 資料結構定義 (Data Structures)

在 C 語言實作中，字元陣列宣告需額外預留 `+1` 空間給字串結束字元 `\0`。

### 2.1 圖書結構 (Book)

每筆圖書資料需包含以下屬性 ：

| 屬性名稱 | C 語言建議型別 | 長度限制 (不含 \0) | 說明 / 範例 |
| --- | --- | --- | --- |
| 書名 (Title) | `char[128]` | 127 字元 | 中英文皆可，例如 `Data Mining: Concepts` |
| 作者 (Author) | `char[50]` | 49 字元 | 中英文皆可，例如 `Deitel & Deitel` |
| 出版年份 (Year) | `char[5]` | 4 字元 | 例如 `2024` |
| 版序 (Edition) | `char[4]` | 3 字元 | 例如 `5` 或 `1` |
| 國際書碼 (ISBN) | `char[14]` | 10 或 13 字元 | 例如 `0131971093` |
| 複本編碼 (CopyId) | `char[4]` | 3 字元 | 例如 `1`, `2` |
| 借閱者 (Borrower) | `char[20]` | N/A | 若無人借閱則為空或特定標記；若借出則為學生學號 

 |

### 2.2 學生資料結構 (Student)

| 屬性名稱 | C 語言建議型別 | 說明 / 範例 |
| --- | --- | --- |
| 帳號 (Account) | `char[20]` | 學生學號，例如 `495713391` 

 |
| 密碼 (Password) | `char[20]` | 預設為學號後四碼，例如 `3391` 

 |
| 借閱清單 (Borrowed) | Array or Linked List | 記錄該學生借閱的多筆圖書，包含 `ISBN` 與 `借閱日期` 

 |

## 3. 檔案 I/O 與解析格式

系統啟動時必須自動讀取以下兩個檔案進行初始化 ：

### 3.1 圖書資料檔 (`Books.txt`)

* **格式說明**：欄位以「空格」分隔。但「書名」與「作者」因為本身可能包含空格，因此被包覆在 `& &` 之間。
* 
**解析邏輯要求**：程式必須能夠正確解析 `&字串內容&`，將其視為單一欄位（去除首尾的 `&`） 。


* 
**範例**：`&How to program C& &Deitel & Deitel& 2024 5 0131971093 2 495713003` (最後一個數字為借閱者學號，若無借閱者則該欄位不存在) 。



### 3.2 學生資料檔 (`Students.txt`)

* 
**格式說明**：`帳號 密碼 [借閱圖書ISBN_1 借閱日期_1] [借閱圖書ISBN_2 借閱日期_2] ...` 。


* 
**解析邏輯要求**：一位學生可能借閱 0 本到多本書。日期格式為 `YYYYMMDD`（如 `20250418`） 。

## 4. 系統指令與功能規格

系統需提供一個選單或命令列輸入介面，支援以下指令 ：

* **指令 0：借書**
1. 提示登入帳號與密碼（驗證失敗需拒絕）。
2. 檢查該學生是否有**任何逾期未還的書籍**（借書期限為 30 天）。若有，不可再借閱其他書籍 。


3. 輸入圖書的「書名」或「ISBN」。
4. 輸入「複本編碼」。
5. 驗證該書是否存在且「未被借出」。若已被借出，需提示無法借閱。
6. 記錄當下日期為借書日期（可使用系統當前日期，或為了符合測試檔，實作一個可手動輸入「今日日期」的除錯選項）。
7. 顯示預計還書日期（借書日 + 30 天）。
8. 詢問是否繼續借書（重複步驟 3~7）直到使用者選擇結束，最後登出 。




* **指令 1：還書**
1. 輸入「書名」或「ISBN」。
2. 輸入「複本編碼」。
3. 將該書狀態改為未借出，並從學生的借閱清單中移除 。




* **指令 2：新增圖書**
1. 依序提示輸入：書名、作者、出版年份、版序、ISBN、複本編碼。
2. 
**限制**：需檢查系統中是否已存在相同的 `ISBN` 與 `複本編碼` 組合。若已存在，需拒絕新增 。




* **指令 3：刪除圖書**
1. 輸入 `ISBN` 與 `複本編碼`。
2. 
**限制**：若該書「正在被借閱中」，不可刪除，需顯示錯誤訊息 。


3. 若未被借出，系統詢問是否確定刪除（Y/N），確認後將其從資料結構中移除 。




* **指令 4：查詢圖書**
1. 提供三種查詢維度：根據「書名」、「作者」或「ISBN」查詢。支援模糊比對或完全比對。
2. 印出符合條件的所有複本，必須包含 7 項資訊：書名、作者、出版年份、版序、ISBN、複本編碼、**是否可借閱 (狀態)** 。




* **指令 5：查詢個人借閱資料**
1. 提示輸入帳號與密碼登入。
2. 印出該帳號目前借閱的所有圖書清單與借閱日期 。




* **指令 6：傾印 (Dump)**
1. 印出系統內「全部圖書」的目前狀態（上述 7 項資訊） 。




* **指令 7：系統結束**
1. 結束程式執行 。

## 5. 核心業務邏輯與驗收測試邊界 (Edge Cases)

請 Agent 在實作時，務必通過以下由「驗收測試指令集」推導出的邊界條件測試 ：

1. 
**逾期阻擋邏輯**：借閱期限嚴格定義為 `30天`。若學生身上有任何一本書借閱天數 $> 30$，則系統必須完全封鎖其「借書」權限，直到該逾期書籍歸還為止（對應測試檔中的借書失敗情境） 。


2. 
**歸還後恢復權限**：一旦逾期書籍被歸還（指令 1），該學生應立即恢復借書權限（對應測試案例 5 與 6 的連續動作） 。


3. 
**重複借閱阻擋**：若一本書（特定 ISBN + 特定複本）已經被任何人借出，即使查詢得到，再次執行借書時也必須顯示錯誤並拒絕（對應測試案例 7） 。


4. **防呆機制 (新增與刪除)**：
* 新增時，若發生 Primary Key 衝突（ISBN + 複本編碼 相同），必須阻擋（對應測試案例 9） 。


* 刪除時，若圖書的狀態為「借出中」，絕對不允許刪除該筆資料（對應測試案例 11） 。




5. **日期計算實作**：請實作一個可靠的日期計算函式，能夠正確計算兩個 `YYYYMMDD` 字串之間的相差天數，並考慮大小月與閏年。
